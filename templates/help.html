Help!

If you somehow do not know how to look for fixing errors in the code, then this site is for you.

Basic search rules:

  • 1. Describe the error.

    Write what type (kind) the error is, in what language and which function caused the error. You can also write an OS, but this is not necessary.

  • 2. Don't write a CD!

    First, each user has a different OS and therefore a CD. And so it turns out that the CD only interferes with the search. Second, the hacker will get the file and directory.

  • 3. Big description doesn't mean good.

    If you just stupidly copy the text of the error, then there is a chance that the error will not be detected. Therefore, it is better to follow the first point.

  • 4. What, no bug fix was found?

    If you could not find a fix for the error, then write about your error in the formulas. Not too late)

After reading, you can go to the main page.
